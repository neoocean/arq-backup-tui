## Two production bugs the GUI restore surfaced (#184)

Invisible to all 20 prior verification surfaces; only manifested in Arq.app's
GUI restore path.

1. **backuprecord path divisor** — records are named by their creation-time
   Unix epoch split `epoch // 10_000_000` (5-digit dir) / `epoch % 10_000_000`
   (7-digit zero-padded file). We used `10**5` + no pad, so GUI restore failed
   `…backuprecord not found` (Arq recomputes the path from `creationDate`).
   Fixed in `arq_writer/backup.py`; `tests/test_backuprecord_numbering.py`
   corrected.
2. **folder name ≠ planUUID** — real Arq names a plan's top-level destination
   folder by its `planUUID`; our default used a separate `computer_uuid`, so
   the GUI didn't recognise the destination. Defaulted them coupled;
   `tests/test_writer_folder_planuuid.py`.
